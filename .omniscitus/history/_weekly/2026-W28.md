## Decisions & Constraints
- No server/DB: localStorage → file-backed store (.zclip-data) — port-proof,
  quota-proof; providers' signed URLs die in days ⇒ clip vault is canonical.
- Refine (≤900 chars, minimal edits) vs SPEC (2–4k chars, full reassembly)
  are separate tracks by design.
- Spec SSOT moved INTO ZCLIP (mono skill retired); spec changes ship only on
  a Spec Lab win (improvement gate); experiment data stays in gitignored /lab.
- Spec interview lives in the composer, persists nothing until Generate —
  deleted a class of edge cases and made live headless testing safe.
- Deterministic rules belong in code, not prompts (one-take⇒cut-board,
  gate-id validation, length backstops).
- Owner UX principles: errors visible; money moves only behind explicit
  confirms; cheapest-capable-model-first for experiments.
